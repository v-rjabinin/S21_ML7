# S21_ML7

Данный проект посвящён методам обучения без учителя, а именно — задаче кластеризации. Он включает в себя сравнительный анализ классических и вероятностных алгоритмов (K-Means, DBSCAN, Agglomerative Clustering, Gaussian Mixture) и оценку качества разбиения внутренними метриками (Silhouette score, Distortion). В проекте также используется техника применения меток кластеров как дополнительных признаков для улучшения качества моделей вместе с их последующей интерпретацией с помощью SHAP на примере данных о рынке аренды недвижимости.

## ⚙️ Реализованные методы и классы

1) Модели кластеризации
- `KMeansS21`: Собственная реализация алгоритма K-Means. Поддерживает различные стратегии инициализации центроидов: случайную (random / Forgy), на основе медианы (median-wise) и на основе среднего (mean-wise). Также включает встроенную визуализацию сходимости алгоритма (график изменения `inertia` по итерациям) и метод `predict` для предсказания кластеров на новых данных;
- `DBSCAN`: Собственная реализация Density-Based Spatial Clustering. Поддерживает различные метрики расстояния (например, manhattan) и настройку гиперпараметров `eps` и `min_samples`.
2) Анализ, валидация и сравнение кластеризации
- `class ClusteringAnalyzer`: Основной класс для комплексного анализа и валидации результатов кластеризации. Включает в себя:
  - Строгую валидацию входных данных (`_validate_x`, `_validate_y`, `_validate_dict`);
  - Расчет метрики искажения (`calculate_distortion`) с корректной обработкой шумовых выбросов (игнорирование точек с меткой $-1$);
  - Расчет Silhouette Score с использованием `sklearn.metrics.silhouette_score`;
  - Визуализацию результатов кластеризации (scatter plot) с гибкой настройкой через словари scatter_params и fig_params;
  - Автоматический отбор наиболее важных признаков (feature importance) с использованием `Lasso` регрессии и подбора гиперпараметров через `GridSearchCV`.
3) Генерация и отбор признаков
- `class FeatureGenerator`: Класс для автоматической генерации новых признаков и борьбы с мультиколлинеарностью. Поддерживает:
  - Нелинейные преобразования: $sin(x)$, $cos(x)$, $ln(x + 1)$, $e^{x}$, $2^x$, $\frac{x}{y}$, $x \cdot y$;
  - Агрегирующие функции (в том числе с группировкой): count, max, min, mean, std, median, 25%, 75%, sum;
  - Полиномиальные признаки;
  - Автоматическое удаление коррелирующих признаков: параметр `drop_corr=True` с настраиваемым порогом корреляции `thld` (по умолчанию $0.9$).

<a id="-основные-задачи-проекта"></a>
## 📈 Основные задачи проекта
1) Реализация и исследование алгоритмов кластеризации
- Разработка собственных версий алгоритмов: `KMeans` и `DBSCAN`;
- Сравнение собственных реализаций с эталонными моделями из `sklearn` на геопространственных данных;
- Построение `k-distance graph` для автоматического и обоснованного подбора параметра `eps` в `DBSCAN`.
2) Автоматизация подбора гиперпараметров
- Оптимизация гиперпараметров для всех алгоритмов (`KMeans`, `DBSCAN`, `AgglomerativeClustering`, `GaussianMixture`) с помощью optuna (`TPEsampler`);
- Обработка математических ограничений алгоритмов на этапе оптимизации (например, принудительная установка `metric='euclidean'` при выборе `linkage='ward'`);
- Разработка пайплайна для сравнения моделей на различных наборах признаков с автоматическим сохранением результатов и бэкапов в csv.
3) Генерация и отбор признаков (*Feature Engineering*)
- Создание модуля `FeatureGenerator` для построения нелинейных признаков;
- Итеративное удаление мультиколлинеарных признаков на основе порогового значения корреляции Пирсона.
4) Решение downstream-задачи и интерпретация моделей
- Использование меток кластеров в качестве дополнительных признаков для обучения `RandomForestRegressor` на данных о рынке аренды недвижимости;
- Интерпретация итоговой модели с помощью библиотеки `SHAP (TreeExplainer)`: анализ глобальной важности признаков, оценка вклада кластерных меток в предсказания и выявление неинформативных разбиений.
